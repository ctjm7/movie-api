See You at The Movies! Documentation

This application retrieves information on movies, directors, and genres. The user can create a profile and save their favorite movies and create a watch list.

Retrieve Movie List

Description: This endpoint retrieves a list of movies as a JSON object.

Endpoint URL: /movies

METHOD Type: GET

Query Parameters: None

Request Body Format: None

Response Body Format: JSON object with list of movies as data

Retrieve Movie Data

Description: This endpoint retrieves a movie with corresponding data including description, director, genre, and image as a JSON object.

Endpoint URL: /movies/:movieTitle

METHOD Type: GET

Query Parameters: :movieTitle

Request Body Format: None

Response Body Format: JSON object with movie and corresponding information

Retrieve a Genre

Description: This endpoint retrieves a genre with description as a JSON object.

Endpoint URL: /movies/genre/:genreName

METHOD Type: GET

Query Parameters: :genreName

Request Body Format: None

Response Body Format: JSON object with genre and corresponding information

Retrieve a Director

Description: This endpoint retrieves a director and biography as a JSON object.

Endpoint URL: /movies/directors/:directorName

METHOD Type: GET

Query Parameters: :directorName

Request Body Format: None

Response Body Format: JSON object with director and corresponding information

Register a User

Description: This endpoint registers the user and returns a JSON object with user name and information along with an automatic generated id.

Endpoint URL: /users

METHOD Type: POST

Query Parameters: None

Request Body Format: JSON object with "name" required

Example:

{
"name": "Laura"
}

Response Body Format: JSON object with user information being stored

Example:

{
"name": "Laura",
"favoriteMovies": [],
"id": "4cb13c06-7b5a-48f2-8ae5-b7e8861dd544"
}

Response Status Codes:

201: JSON object with user information

400: user needs name

Update User Information

Description: This endpoint updates the user information and returns a JSON object with user name and information along with an automatic generated id.

Endpoint URL: /users/:id

METHOD Type: PUT

Query Parameters: :id

Request Body Format: JSON object with new user information

Example:

{
"name": "Hannah"
}

Response Body Format: JSON object with updated user information

Example:

{
"id": 1,
"name": "Hannah",
"favoriteMovies": []
}

Response Status Codes:

201: JSON object with user information

400: user not found

Add a Movie to Favorites List

Description: This endpoint adds a movie to the user's favoirite movie list. It returns a text response with movie title and user id.

Endpoint URL: /users/:id/:movieTitle

METHOD Type: POST

Query Parameters: :id, :movieTitle

Example: /users/2/New%20Movie

Request Body Format: None

Response Body Format: Text with movie and user id

Example: New Movie has been added to user 2

Response Status Codes:

201: generated text response

400: user not found

Delete a Movie from Favorites List

Description: This endpoint deletes a movie from the user's favoirite movie list. It returns a text response with movie title and user id.

Endpoint URL: /users/:id/:movieTitle

METHOD Type: DELETE

Query Parameters: :id, :movieTitle

Example: /users/2/New%20Movie

Request Body Format: None

Response Body Format: Text with movie and user id

Example: New Movie has been removed from user 2

Response Status Codes:

201: generated text response

400: user not found

Delete a User

Description: This endpoint removes a user.

Endpoint URL: /users/:id

METHOD Type: DELETE

Query Parameters: :id

Example: /users/2/New%20Movie

Request Body Format: None

Response Body Format: Text with user id

Example: user 3 has been deleted

Response Status Codes:

201: generated text response

400: user not found
