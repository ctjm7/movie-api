See You at The Movies! Documentation

This application retrieves information on movies, directors, and genres. The user can create a profile and save their favorite movies and create a watch list.

Retrieve Movie List

Description: This endpoint retrieves a list of movies as a JSON object.

Endpoint URL: /movies

METHOD Type: GET

Query Parameters: None

Request Body Format: None

Response Body Format: JSON object with list of movies as data

Retrieve Movie Data

Description: This endpoint retrieves a movie with corresponding data including description, director, genre, and image as a JSON object.

Endpoint URL: /movies/:Title

METHOD Type: GET

Query Parameters: :Title

Request Body Format: None

Response Body Format: JSON object with movie and corresponding information

Retrieve a Genre

Description: This endpoint retrieves a genre with description as a JSON object.

Endpoint URL: /movies/genre/:Name

METHOD Type: GET

Query Parameters: :Name

Example: /movies/genre/Comedy

Request Body Format: None

Response Body Format: JSON object with genre and corresponding information

Retrieve a Director

Description: This endpoint retrieves a director and biography as a JSON object.

Endpoint URL: /movies/directors/:Name

METHOD Type: GET

Query Parameters: :Name

Example: /movies/directors/Rob%20Reiner

Request Body Format: None

Response Body Format: JSON object with director and corresponding information

Register a User

Description: This endpoint registers the user and returns a JSON object with user name and information along with an automatic generated id.

Endpoint URL: /users

METHOD Type: POST

Query Parameters: None

Request Body Format: JSON object with username, password, and email required

Example:

{
"Username": "Joe",
"Password": "toocool",
"Email": "jj@gmail.com",
"Birthday": new Date("1990-09-21"),
}

Response Body Format: JSON object with user information being stored

Example:

{
"_id": "63042df019706ddd090db11d", "Username": "Joe", "Password": "toocool", "Email": "jj@gmail.com", "Birthday": "1990-09-21", "FavoriteMovies": [] }

Response Status Codes:

201: JSON object with user information

400: Text response (:Username already exists)

500: 'Error'

Update User Information

Description: This endpoint updates the user information and returns a JSON object with username and information.

Endpoint URL: /users/:Username

METHOD Type: PUT

Query Parameters: :Username

Request Body Format: JSON object with new user information with username required

Example:

{
"Username": "Herb",
"Email": "hrby@gmail.com"
}

Response Body Format: JSON object with updated user information

Example:

{
"_id": "63042df019706ddd090db11d", "Username": "Herb",
"Password": "yty99",
"Email": "hrby@gmail.com",
"FavoriteMovies": []
}

Response Status Codes:

201: JSON object with user information

500: 'Error'

Add a Movie to Favorites List

Description: This endpoint adds a movie to the user's favorite movie list. It returns a JSON object with the updated user information.

Endpoint URL: /users/:Username/movies/:MovieID

METHOD Type: POST

Query Parameters: :Username, :MovieID

Example: /users/Herb/movies/63012c7f5fd67ec6a36255de

Request Body Format: None

Response Body Format: JSON object with updated user information

Example: {
"_id": "63042df019706ddd090db11d",
"Username": "Herb",
"Password": "yty99",
"Email": "hrby@gmail.com",
"FavoriteMovies": ["63012c7f5fd67ec6a36255de"]
}

Response Status Codes:

201: JSON with updated user information

500: 'Error'

Delete a Movie from Favorites List

Description: This endpoint deletes a movie from the user's favorite movie list. It returns a JSON object.

Endpoint URL: /users/:Username/movies/:MovieID

METHOD Type: DELETE

Query Parameters: :Username, :MovieID

Example: /users/Herb/movies/63012c7f5fd67ec6a36255de

Request Body Format: None

Response Body Format:

Example:

{
"_id": "63042df019706ddd090db11d",
"Username": "Herb",
"Password": "yty99",
"Email": "hrby@gmail.com",
"FavoriteMovies": []
}

Response Status Codes:

201: JSON object with updated user information

500: 'Error'

Delete a User

Description: This endpoint removes a user.

Endpoint URL: /users/:Username

METHOD Type: DELETE

Query Parameters: :Username

Example: /users/Herb

Request Body Format: None

Response Body Format: text response

Example: Herb was deleted

Response Status Codes:

201: Text response (:Username was deleted)

400: Text response (:Username was not found)

500: 'Error'
